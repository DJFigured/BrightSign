## 1. ANALYZA AKTUALNICH BEST PRACTICES

### 1.1 Co B2B zakaznici ocekavaji (data z pruzkumu 2025-2026)
- **60% B2B zakazniku** opusti stranku pokud se nacita pomalu
- **78% chce lepsi mobilni zazitek** nez maji soucasni dodavatele
- **27% rika** ze snadne vyhledavani produktu je nejdulezitejsi
- **32% rika** ze jednoduchy checkout je klicovy

### 1.2 Klicove UX principy pro B2B e-shop
1. **Trust first** -- duveryhodnost pred designem (reference, certifikace, konkretni cisla)
2. **Search-driven** -- B2B zakaznici vedi co chteji, hledaji konkretni model
3. **Specifications matter** -- technicke detaily musi byt rychle pristupne
4. **Price transparency** -- ceny bez DPH + s DPH, B2B slevy viditene
5. **Speed** -- stranky se musi nacitat pod 2 sekundy
6. **Documentation** -- datasheety, manualky, SDK ke stazeni

## 3. UX PRINCIPY PRO BRIGHTSIGHT

### 3.1 Hlavni principy

1. **Profesionalita nad kreativitou**
   - Cily: zakaznik musi okamzite vedet ze nakupuje od seriozni firmy
   - Provedeni: konzistentni fotky, cisty layout, zadne zbytecne animace

2. **Informace pred estetikou**
   - B2B zakaznik hledá: model, cenu, specifikace, dostupnost, dodaci lhut
   - Vsechny tyto informace musi byt viditele BEZ scrollovani na produktove strance

3. **Duvera pres dukazy**
   - Konkretni cisla: "500+ instalaci v CR", "200+ B2B zakazniku"
   - Partner badge, certifikace, reference
   - Telefonni cislo viditene v headeru

4. **Dva rezimy pro dva zakazniky**
   - B2C: ceny s DPH, jednoduchy checkout, rychly nakup
   - B2B: ceny bez DPH, firemni udaje, fakturace, slevove urovne

5. **Mobile-first ale desktop-optimized**
   - Vetsina B2B traffic je z desktopu (70%+)
   - Ale mobilni verze musi byt bezchybna pro hledani a prohlizeni

### 3.2 Vizualni jazyk

- **Barvy:** Ponechat brand-primary (#1a2b4a) + brand-accent (#00c389) -- souhlasi s BrightSign brandem
- **Typografie:** Inter je spravna volba -- citelny, profesionalni, moderni
- **Ikonka:** Lucide icons -- konzistentni, ciste
- **Fotky:** Bile pozadi, konzistentni uhel, vysoke rozliseni
- **Spacing:** Vic whitespace nez ted -- stranky pusobi trochu stisnene

## 6. TECHNICKE POZNAMKY PRO SUBAGENTA

### Existujici stack (nemenit):
- Next.js 15 + next-intl
- Tailwind CSS s brand-* tokeny
- Lucide icons
- shadcn/ui komponenty (Button, Card, Badge, Input, atd.)
- Medusa.js SDK pro data

### Lokalizovane URL:
- Stranky maji ceske URL (/hledani, /kategorie, /porovnani)
- To je v poradku pro MVP, ale POZDEJI by bylo idealni mit lokalizovane URL pro kazdy jazyk
- Nepremenovavat ted -- prilis velky refactor

### Layout:
- Header ma 3 urovne: top bar, main nav, category nav
- Footer ma 5 sloupcu: 2x company+newsletter, produkty, odkazy, kontakt+platby
- Obe jsou solidni zaklad, jen potrebuji vizualni vylepseni

### Responsivita:
- Grid 2 sloupce na mobilu, 4 na desktopu pro produkty
- Mobile menu jako Sheet (slide-in)
- Search je toggle na mobilu
- Vsechno funguje, jen potrebuje lepe vypadat
